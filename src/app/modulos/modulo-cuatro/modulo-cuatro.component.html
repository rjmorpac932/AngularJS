4.2 - Navegación entre componentes
4.2.1 - Uso de enlaces y rutas

La navegación en una aplicación web principalmente se realiza desde la misma aplicación haciendo clicks sobre enlaces a otras vistas.

En HTML, estos enlaces se realizan mediante la etiqueta y su atributo “href”, que sería donde pondríamos la dirección URL. Sin embargo, al haber creado un servicio Router, nos interesa que todo el control de rutas lo realice el mismo servicio. Por eso Angular dispone de las directivas RouterLink.

Las directivas RouterLink nos permiten crear enlaces a partes específicas de nuestras aplicaciones.

Si en las etiquetas sustituimos los atributos “href” por directivas RouterLink, conseguiremos que el servicio Router tome el control de todos estos elementos. Los path configurados en RouterLink pueden ser estáticos:

O también pueden ser dinámicos, donde usamos una cadena con literales y variables para formar el path dinámicamente:

En el primer ejemplo, se hace uso de la directiva routerLinkActive. Esta directiva sirve para aplicar una clase CSS al elemento cuando la ruta del link esté activa. De esta manera podremos distinguir qué enlace, de todos los visibles en la aplicación, es el que está activo.

Por otra parte, el primer segmento del path puede venir precedido por “/”, “../” o “./”. Esto indica a Angular en qué parte del árbol de rutas realizar el link del path. Si, por el URL activo, el que aparece en la barra de direcciones, fuese “localhost:4200/libros” y quisiéramos desplazarnos a “localhost:4200/libros/comics”, en routeLink podríamos poner el path “/libros/comics” o “./comics” o “comics”.

También se puede hacer un path con **, lo que funciona como ruta comodín. El asterisco en las rutas sirve para especificar que éstas coincidan con cualquier ruta que comience con la parte especificada antes del asterisco, lo que significa que una ruta con ** es especialmente útil para capturar cualquier ruta no coincidente. Para obtener el valor de los parámetros que pueden venir por URL, el componente utiliza el objeto ActivatedRoute. ActivatedRoute contiene información de la ruta asociada al componente que actualmente tengamos cargado en la etiqueta “router-outlet”, por lo tanto, información de la actual ruta activa. Con el uso de la etiqueta router-outlet ya no será necesario llamar (ni importar) a cada componente sino que podremos acceder a cada uno según como se haya configurado en las rutas.

A continuación vamos a ver los parámetros para configurar las rutas:

Obligatorios

  • path: Sirve para indicar la parte de la URL que activará un componente específico.
  • component: Sirve para indicar el componente que será llamado con una URL específica.

Opcionales

  • data: Sirve para almacenar metadatos asociados a la ruta.
  • title: Sirve para definir el título de una página.
  • redirectTo: Sirve para redireccionar al usuario a otra ruta.
  • pathMatch: Sirve para definir cómo debería coincidir la URL con la ruta definida con redirectTo (full/prefix).
